Executive Summary

CV Analyzer is an LLM-powered application that performs semantic analysis between resumes and job descriptions, providing quantitative matching scores and actionable insights.

Key Capabilities

  • Intelligent Document Processing: Gemini Vision API for advanced CV parsing.
  • Semantic Analysis: LLM-based deep understanding of skills & experience.
  • Structured Output: JSON-formatted analysis with clear scoring metrics.
  • Scalable Architecture: Stateless backend with external LLM API integration.

Request Flow

  1. 1
    Upload: User uploads a resume via the drag-and-drop interface.
  2. 2
    Process: The backend orchestrates the file handling and parsing workflow.
  3. 3
    Parse: Gemini Vision API extracts structured JSON data from the resume.
  4. 4
    Analyze: Gemini 2.0 API performs semantic matching against a selected job description.
  5. 5
    Store: Analysis results are cached in the database for performance and history.
  6. 6
    Display: The UI presents scores and actionable insights to the hiring manager.

Prompt Engineering: The Intelligence Layer

Our system's precision comes from meticulously crafted, multi-stage prompts. We enforce a strict JSON-only output format to ensure reliable, structured data for downstream processing.

CV Parsing Prompt Strategy


# This prompt instructs Gemini Vision to act as an expert parser.
def create_extraction_prompt():
    return """
    You are an expert CV/Resume parser. Extract ALL information 
    from the provided document and return it as a valid JSON object.

    **IMPORTANT INSTRUCTIONS:**
    1. Return ONLY valid JSON - no markdown, no explanations.
    2. Use null for missing optional fields.
    3. Categorize technical skills appropriately.
    """

Analysis Prompt Strategy

For analysis, we use a chain of prompts to evaluate different facets of the candidate's profile, ensuring a comprehensive and nuanced evaluation.

Analysis Module & Scoring Algorithm

The Analysis module aggregates scores from the multi-stage prompt chain into a single, weighted suitability score, providing a clear, top-level metric for decision-making.

Technical Differentiators

Our architecture is designed for performance, scalability, and reliability, setting it apart from simpler implementations.

Why This Architecture?

  • Dual LLM Strategy: Using the best model for each task (Vision for parsing, Text for analysis).
  • Async-First Design: FastAPI allows for high-concurrency I/O operations, crucial for API calls.
  • Repository Pattern: Decouples business logic from data access for easier testing and maintenance.

Key Implementation Details

  • Pydantic Models: Enforces strict data validation at the API boundary.
  • Robust Retry Logic: Implements exponential backoff for external API calls to handle transient errors.
  • Flexible JSON Storage: Allows for easy schema evolution as prompt outputs change.

Security Considerations

  • PII Handling: CVs are treated as sensitive data and are not stored long-term unless required.
  • API Security: API keys are managed via environment variables and are not exposed in code.
  • Access Control: Future iterations will include role-based access for multi-user environments.
